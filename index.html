01 — Ready-to-Use Templates

Drop these into Claude Cowork and start producing immediately.

Each template includes a copy-paste prompt and step-by-step instructions for your team. These are the exact templates we use daily. Click to expand.

How to Use
Create a new project in Claude Cowork called "Content Engine — [Topic/Campaign Name]".
Upload your brand voice doc, audience personas, and any existing top-performing content as project knowledge.
Paste the prompt below into the project chat.
Review the output. Ask Claude to expand any section — e.g. "Write the full LinkedIn post for angle #3."
Use the project as a living workspace — add team members so writers, designers, and social managers can all pull from the same source.
Copy-Paste Prompt
You are a senior content strategist. I need a comprehensive multi-channel content plan. **CORE TOPIC:** [Insert your topic or key message] **TARGET AUDIENCE:** [Insert audience — e.g. "B2B SaaS founders, Series A-C, 20-200 employees"] **GOAL:** [Insert goal — e.g. "Drive demo bookings" / "Build thought leadership" / "Launch product awareness"] **BRAND VOICE:** [Insert 2-3 adjectives — e.g. "Bold, practical, zero-fluff"] --- Please generate the following: ## 1. Content Angles (5-7 unique angles) For each angle provide: - Hook (1 sentence that stops the scroll) - Core argument - Target emotion - Best channel for this angle ## 2. Blog Post Outlines (3 posts) For each: - SEO-optimized title (include primary keyword) - Meta description (155 chars) - H2 structure with key points under each - Internal/external linking suggestions - CTA recommendation ## 3. LinkedIn Posts (5 posts) For each: - Full post text (hook + body + CTA) - Suggested posting day/time - Engagement prompt (question or poll idea) ## 4. Email Sequence (3-email nurture) For each email: - Subject line (A/B options) - Preview text - Full body copy - CTA button text ## 5. Twitter/X Thread (1 thread, 8-12 tweets) - Full thread with numbering - Include a visual/screenshot suggestion for tweet 1 ## 6. Short-Form Video Scripts (2 scripts) - 60-second format - Hook (first 3 seconds), body, CTA - On-screen text suggestions Ensure all content is consistent in messaging but adapted to each channel's native format.
💡 Pro Tip: After generating, tell Claude "Now create a content calendar for the next 4 weeks using all of the above." Since it has the full context in the project, it will schedule everything — including which pieces support which.
How to Use
Create a project called "Product Roadmap — [Quarter/Year]".
Upload: customer feedback docs, support ticket summaries, competitor analysis, sales call notes, and any existing roadmap.
Invite your PM, engineering lead, and design lead to the project so everyone can iterate together.

Copy-Paste Prompt
You are a senior product manager. Using all the context in this project (customer feedback, support data, competitor intel, sales notes), build a quarterly product roadmap. **QUARTER:** [e.g. Q3 2025] **TEAM SIZE:** [e.g. "3 engineers, 1 designer, 1 PM"] **COMPANY STAGE:** [e.g. "Series B, 5K customers, PLG motion"] **NORTH STAR METRIC:** [e.g. "Weekly active teams" / "Net revenue retention"] --- Please generate: ## 1. Signal Synthesis - Top 10 themes from customer feedback (ranked by frequency + revenue impact) - Top 5 competitive gaps - Top 3 internal pain points from the team ## 2. Opportunity Scoring For each of the top 10 themes, score on: - Impact (1-10): How much does this move the north star metric? - Confidence (1-10): How sure are we this matters? - Effort (T-shirt size): XS / S / M / L / XL - Calculate ICE score and rank ## 3. Recommended Roadmap Organize into: - **Must-haves** (non-negotiable this quarter) - **Should-haves** (high impact, schedule permitting) - **Nice-to-haves** (opportunistic wins) - **Explicitly NOT doing** (and why — this is critical for alignment) For each item: - 1-paragraph scope description - Success metric - Dependencies - Estimated timeline (weeks) ## 4. Sprint Breakdown - Suggested 2-week sprint plan for the first 6 weeks - Key milestones and review points ## 5. Stakeholder Summary - 1-page executive summary suitable for a board update - 3 talking points for the sales team - 3 talking points for customer success
💡 Pro Tip: Keep this project alive all quarter. After each sprint, paste your retro notes and ask Claude to "Update the roadmap based on what we learned." The persistent context means it gets smarter every iteration.
How to Use
Create a project called "Research Synthesis — [Research Initiative Name]".
Upload all raw data: interview transcripts, survey CSVs, NPS comments, support tickets, Gong recordings (transcripts), G2 reviews.
Paste the prompt below. Claude will process everything in the project context.
Ask follow-ups like "Deep dive into the onboarding friction theme" or "Create a persona based on Segment 2."
Share the project with your team — anyone can ask Claude questions about the research data directly.
Copy-Paste Prompt
You are a senior UX researcher and strategist. Analyze all the customer research data in this project and produce a comprehensive synthesis. **RESEARCH GOAL:** [e.g. "Understand why trial-to-paid conversion dropped 15% in Q2"] **CUSTOMER SEGMENT:** [e.g. "Mid-market teams, 50-500 employees"] **DATA SOURCES IN THIS PROJECT:** [List what you uploaded] --- Please generate: ## 1. Executive Summary - 3-paragraph overview a CEO can read in 2 minutes - Top 3 findings with confidence levels (High/Medium/Low) - Recommended immediate actions ## 2. Theme Analysis Identify the top 8-10 themes. For each: - Theme name and 1-sentence description - Frequency (how many data points support this) - Severity (how much it impacts the user experience, 1-5) - Representative quotes (2-3 verbatim quotes per theme) - Affected user segments ## 3. User Segments - Identify 3-4 distinct behavioral segments from the data - For each: demographics, goals, frustrations, current workarounds - Which segment is highest priority and why ## 4. Journey Map - Map the current user journey from awareness → activation → retention - Identify friction points at each stage (backed by data) - Emotional state at each stage ## 5. Competitive Context - Where did users mention alternatives? What were they? - What are users getting from competitors that they're not getting from us? ## 6. Recommendations - Prioritized list of 5-7 recommendations - For each: what to do, expected impact, effort estimate, and supporting evidence - Quick wins (< 1 week) vs. strategic bets (1+ month) ## 7. Appendix - Full quote bank organized by theme - Data methodology notes - Gaps in the research (what we still don't know)
💡 Pro Tip: After synthesis, ask "Create a 10-slide presentation deck summarizing these findings for the leadership team." Claude will structure the full deck with speaker notes since it has all the context.

Copy-Paste Prompt
You are an award-winning creative strategist. Generate a comprehensive campaign brainstorm based on the brief below and all project context. **CAMPAIGN OBJECTIVE:** [e.g. "Launch new enterprise tier — drive 200 qualified demos in 60 days"] **TARGET AUDIENCE:** [e.g. "VP Engineering and CTO at companies with 200-2000 employees"] **KEY MESSAGE:** [e.g. "The only AI workspace built for teams, not individuals"] **BUDGET RANGE:** [e.g. "$50K-$100K"] **CHANNELS AVAILABLE:** [e.g. "LinkedIn Ads, Google Ads, Email, Webinars, Content, Partnerships"] **TIMELINE:** [e.g. "8 weeks"] --- Please generate: ## 1. Strategic Foundation - Campaign positioning statement (1 sentence) - 3 core value propositions (ranked by uniqueness) - Emotional driver analysis: What feeling drives the purchase decision? - Messaging hierarchy: Primary → Secondary → Tertiary messages ## 2. Campaign Concepts (Generate 5 distinct concepts) For each concept: - Concept name (catchy, memorable) - Big idea (1 sentence) - Tagline options (3 per concept) - Hero visual description - Channel strategy (which channels, why) - Content pieces needed (list specific deliverables) - Estimated budget split - Why this works (strategic rationale) ## 3. Messaging Variants For the top concept, create: - Awareness-stage messaging (cold audience) - Consideration-stage messaging (warm audience) - Decision-stage messaging (hot leads) - Each with: headline, body copy, CTA ## 4. Content Calendar - Week-by-week breakdown for the campaign duration - Specific deliverables per week - Dependencies and deadlines ## 5. Measurement Framework - KPIs per channel - Weekly targets - Attribution model recommendation ## 6. Risk Assessment - Top 3 risks to campaign success - Mitigation plan for each - Plan B if primary concept underperforms by week 3
💡 Pro Tip: After the brainstorm, ask "Now write the creative brief for our designer based on Concept #[X]." Since Claude has all the context, it will produce a brief that actually contains everything the designer needs — no back-and-forth.
How to Use
Create a persistent project called "Team Syncs — [Team Name]" that lives across weeks.
Before each meeting: team members paste their async updates into the project chat (what they shipped, what's blocked, what's next).
Run the Pre-Meeting prompt 30 minutes before the sync.
Use the generated agenda to run a focused 30-minute meeting instead of a 60-minute ramble.
After the meeting, run the Post-Meeting prompt to generate action items and update the project history.
Pre-Meeting Prompt
Based on all async updates shared in this project this week, generate: ## 1. Meeting Agenda (30 minutes max) - Pre-populated with the 3-5 most important discussion topics - For each: context summary, key decision needed, time allocation - Flag any cross-team dependencies or blockers ## 2. Progress Snapshot - What shipped this week (celebrate wins) - What's in progress (expected completion) - What's blocked (and suggested unblock actions) ## 3. Metrics Update - Pull any numbers mentioned in updates - Compare to last week's targets (if available in project history) - Flag anything off-track ## 4. Decisions Needed - List decisions that require live discussion (can't be async) - For each: provide context, options, and a recommended path Keep the agenda tight. If it can be decided async, flag it as "async decision" and suggest who should own it.
Post-Meeting Prompt
Based on the meeting discussion above, generate: ## 1. Meeting Summary (3-5 bullet points) - Key decisions made - Important context shared ## 2. Action Items For each action item: - What needs to be done (specific and measurable) - Who owns it - Deadline - Dependencies ## 3. Parking Lot - Topics raised but not resolved - Suggested async follow-up plan ## 4. Next Week Preview - Based on this week's progress, what should next week's priorities be? - Any upcoming deadlines or milestones to flag Update the project context with these decisions so they carry forward to next week's sync.
💡 Pro Tip: The magic is persistence. By week 4, Claude has a full history of your team's velocity, recurring blockers, and decision patterns. Ask it "What patterns do you see in our blockers over the past month?" and watch it surface insights no human would have tracked.

04 — 7-Day Implementation Plan

Go from zero to fully operational in one week.

Follow the steps, ship the system. By Day 7, your entire team is collaborating with AI — not just using it as a solo search engine.

Day 1 — Foundation
Set Up Your Workspace Architecture
Build the project structure your team will live in.
✓Create your Claude Cowork team workspace
✓Create 4 core projects: "Content Engine", "Product Roadmap", "Research Hub", "Team Syncs"
✓Upload brand voice doc, audience personas, and company overview to each relevant project
✓Invite your team — assign at least 2 people per project to start
Day 2 — Context Loading
Feed Claude Everything It Needs to Be Useful
This is the step most teams skip — and why their AI results are mediocre.
✓Upload your top 10 performing content pieces to the Content Engine project
✓Upload customer research, support ticket exports, and NPS data to the Research Hub
✓Upload existing product roadmap, feature specs, and sprint retros to Product Roadmap
✓Add competitor analysis docs and positioning to relevant projects
✓Test each project with a simple question to verify Claude has absorbed the context
Day 3 — First Templates
Run Your First Two Templates on Real Work
Don't practice on fake projects — use real work. You'll see value immediately.
✓Run Template 1 (Content Strategy Engine) on an upcoming launch or campaign
✓Run Template 5 (Weekly Team Sync) for your next team meeting
✓Have each team member interact with the project — ask follow-ups, request iterations
✓Document what surprised you (good and bad) — this informs Day 5 optimization
Day 4 — Deeper Workflows
Run the Remaining Templates
Now that the team has muscle memory, tackle the complex ones.
✓Run Template 2 (Product Roadmap Builder) with your PM and eng lead
✓Run Template 3 (Customer Research Synthesizer) on any existing research data
✓Run Template 4 (Campaign Brainstorm) with cross-functional team present
✓Notice how the projects start connecting — research informs product informs content
Day 5 — Optimize
Customize Templates to Your Team
Adapt the system to your specific needs and language.
✓Modify prompts based on Day 3-4 learnings — add your terminology, adjust output formats
✓Create 1-2 custom templates for workflows unique to your team
✓Set up project naming conventions and folder structure for scale
✓Identify which team members are power users — they'll train the rest
Day 6 — Integrate
Connect Cowork to Your Existing Stack
Make Claude Cowork the hub, not another silo.
✓Establish a workflow for moving outputs from Cowork → your publishing/project tools
✓Set up a cadence: when does the team update project context? (Weekly? After each sprint?)
✓Create a "New Hire Onboarding" project so new team members get up to speed via Claude
✓Define what stays in Cowork vs. what lives in other tools
Day 7 — Launch
You're Operational. Now Scale.
The system is live. Here's how to keep the momentum.
✓Run your first fully AI-augmented team sync using Template 5
✓Measure: count hours saved this week vs. last week on equivalent work
✓Share wins internally — nothing drives adoption like visible results
